Basic Concepts

  • Object-Oriented Programming

    This means using code to simulateTo model or imitate real-life objects and their behaviors.

    For example, consider a cup. A cup has certain propertiesThings that belong to it, such as its color and the amount of liquid it can hold. You can perform certain operations on a cup, such as adding and removing liquid from it (meaning filling it up or drinking from it). There are also certain things about a cup that are variableSomething that may or may not change, such as the amount of liquid it contains at a given time.

  • Variable

    This is basically a name that referencesPoints to a place where a piece of information is stored.

    For example, imagine you have a school binder with labeled sections for Math and Science. You might change the paper in either section, but you know which divider takes you to which notes. In this example, Math and Science are like variables and the school notes they contain are the specific information they reference. This information might change as you add or remove items, but the name on the divider always tells you where your first page of notes for that subject will be.

  • Computational Thinking

    Sound confusing? This is really a fancy way of saying "breaking a problem down into individual steps that, when followed sequentiallyIn order, produce a desiredA thing you want result". For example, if you're giving someone directions from point A to point B, you might say something like "walk two blocks, turn right, walk one block, turn left, and it's right in front of you; if the person doesn't follow these instructions in order, they aren't going to end up in the right place. And that sequence of instructions you follow? We call that an algorithmA formula or set of instructions you follow to complete a task.

    An important part of computational thinking is recognizing when you're doing the same thing over again; in other words, finding patterns and routines that use the same common algorithm. For example, you follow an algorithm when you brush your teeth! You use new toothpaste every time you brush your teeth, and you replace your toothbrush every few months, but the process you follow always stays the same.

  • Computer Programs

    Most computer programs follow a three-step process:

    1. Receive Input
    2. Process Input
    3. Produce Output

    For example, consider the math problem "1 + 2 = 3". The input is "1 + 2", the processing is the process of performing the math given by the input, and then the output is the result (3). When we write computer programs, our goal is to make sure that our process is correct.

Conditionals

  • Boolean Logic

    The word boolean is just a fancy way of saying "something that is true or false". For example, whether or not something is an apple is a boolean condition, because something is either an apple or it's not. We use boolean logic in programming to decide when to do something.

  • Conditional Statements

    This is a way of coding a response to some boolean condition. For example, if an object is an apple, we want to take a bite out of it; if the object is not an apple (maybe it's a chair), then we may or may not want to choose to do something else.

    The most basic conditional statement in programming is the if-else statement. Let's say we have a variable named my_condition; in python, we could write the following conditional statements: 

    if my_condition:
    # This happens when my_condition is true
    print("I'm true!")
    if my_condition:
    # This happens when my_condition is true
    print("I'm true!")
else:
    # This happens when my_condition is false
    print("I'm false!")
    Now, let's say we have multiple things we want to check, like in our apple scenario. We can add an else-if statement between our if and our else: 
    if is_apple(my_object):
    # Write code here to take a bite
elif is_chair(my_object):
    # Write code here to sit in it
else:
    print("The object is not an apple or a chair.").

    It's important to notice the following rules:

    • The first conditional is always if.
    • Any conditionals between an if and an else should be else-if statements.
    • The else statement will only run if everything above it evaluates to false.
    • The else statement is optional.
    • Only one statement in a group of chained conditional statements will ever run.
